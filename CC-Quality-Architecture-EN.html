Analysis Engine Details
5 Engines, Each Measuring a Different Quality Dimension
1
Rule Compliance Engine
Extracts rules from CLAUDE.md as imperative sentences (regex: lines starting with "Always…", "Never…", "Use…", "- "). For each rule, searches for violation evidence throughout the session. Example: if "Always use TypeScript" exists but a .js file is created → violation.
Output Metric compliance_rate = obeyed_rules / total_rules × 100
2
Retry Loop Detector
Two detectors run: (1) Same tool + similar arguments across 3+ consecutive tool calls — a real loop. (2) Same error message appearing 2+ times in tool results — each repetition counted as a separate event. Token cost is event-based, not span-based: a repeated error at turns 5 and 580 does not count the 575 turns between them as waste.
Output Metric retry_waste = Σ(loop_tokens) / total_tokens × 100
3
Attention Dead Zone Mapper
Divides CLAUDE.md into 5 zones (top 20%, upper-middle, middle, lower-middle, bottom 20%). Statistically compares the violation rate of rules in each zone. Validates the LLM "lost in the middle" tendency against real session data.
Output Metric dead_zone_ratio = mid_violations / edge_violations
4
Re-Read Cost Calculator
Groups all Read tool calls by file path. The first read is classified as "necessary," subsequent ones as "redundant." CLAUDE.md special case: since it is re-injected every turn, a multiplier effect is calculated (e.g., 6738% overhead observed in real sessions).
Output Metric reread_overhead = redundant_read_tokens / unique_read_tokens × 100
5
Waste Classifier
Assigns each token to one of 6 categories: Productive (actual work), Re-Read (redundant reads), Retry (loops), System (CLAUDE.md + tool schema overhead), Compaction (summarization cost), Cache-Miss (non-cached input). Performs Pareto analysis. Note: Cache-Miss is not cumulative with the others — it is a cross-cutting efficiency indicator, not added to total waste.
Output Metric efficiency = productive_tokens / total_tokens × 100
Σ
Session Quality Index (SQI)
Weighted composite of 5 metrics: Compliance (w:0.25), Read Efficiency (w:0.25), Retry Efficiency (w:0.20), Attention Distribution (w:0.15), Token Utilization (w:0.15). Each normalized to 0–100, then weighted average. Multiplicative penalty: when loop_count > 10 or retry_waste > 20%, the result is multiplied by ×0.85 — a high loop count cannot hide behind good scores elsewhere.
Composite Formula SQI = Σ(wᵢ × scoreᵢ) × penalty → 0–100
penalty = 0.85 if loop_count>10 ∨ retry_waste>20%

Data Structure
JSONL Line Schema (Claude Code)
// ~/.claude/projects/<project>/session-<id>.jsonl — each line:
{
  "type": "assistant" | "user" | "system",
  "message": {
    "role": string,
    "content": [
      { "type": "text", "text": string },
      { "type": "tool_use", "name": "Read", "input": { "file_path": string } },
      { "type": "tool_result", "content": string }
    ]
  },
  "usage": {
    "input_tokens": number,  // ← CC Quality's primary data source
    "output_tokens": number,
    "cache_read_input_tokens": number,
    "cache_creation_input_tokens": number
  },
  "timestamp": ISO 8601,
  "sessionId": string
}
