Analiz Motorları Detayı
5 Motor, Her Biri Farklı Bir Kalite Boyutunu Ölçer
1
Rule Compliance Engine
CLAUDE.md'den kuralları imperative cümle olarak çıkarır (regex: satır başı "Always…", "Never…", "Use…", "- " ile başlayan direktifler). Her kural için session boyunca ihlal kanıtı arar. Örnek: "Always use TypeScript" kuralı varken .js dosyası oluşturulması → ihlal.
Çıktı Metriği compliance_rate = obeyed_rules / total_rules × 100
2
Retry Loop Detector
İki dedektör çalışır: (1) Ardışık 3+ tool call'da aynı araç + benzer argüman — gerçek döngü. (2) Aynı hata mesajının tool result'larında 2+ kez tekrarlanması — her tekrar ayrı olay olarak sayılır. Token maliyeti span tabanlı değil, olay tabanlıdır: 5. ve 580. turda tekrarlayan hata ikisi arasındaki 575 turu waste olarak saymaz.
Çıktı Metriği retry_waste = Σ(loop_tokens) / total_tokens × 100
3
Attention Dead Zone Mapper
CLAUDE.md'yi 5 bölgeye ayırır (üst %20, üst-orta, orta, alt-orta, alt %20). Her bölgedeki kuralların ihlal oranını istatistiksel olarak karşılaştırır. LLM'lerin "lost in the middle" eğilimini session verisiyle doğrular.
Çıktı Metriği dead_zone_ratio = mid_violations / edge_violations
4
Re-Read Cost Calculator
Tüm Read tool çağrılarını file path'e göre gruplar. İlk okuma "gerekli", sonrakiler "gereksiz" olarak sınıflandırılır. CLAUDE.md özel durum: her turn'de yeniden enjekte edildiği için çarpan etkisi hesaplanır (senin bulduğun %6738 gibi).
Çıktı Metriği reread_overhead = redundant_read_tokens / unique_read_tokens × 100
5
Waste Classifier
Her token'ı 6 kategoriye atar: Productive (asıl iş), Re-Read (gereksiz okuma), Retry (döngü), System (CLAUDE.md + tool schema overhead), Compaction (özetleme maliyeti), Cache-Miss (cache dışı input). Pareto analizi yapar. Not: Cache-Miss diğerleriyle kümülatif değildir — çapraz bir verimlilik göstergesidir, toplam waste'e eklenmez.
Çıktı Metriği efficiency = productive_tokens / total_tokens × 100
Σ
Session Quality Index (SQI)
5 metriğin ağırlıklı bileşimi: Compliance (w:0.25), Read Efficiency (w:0.25), Retry Efficiency (w:0.20), Attention Distribution (w:0.15), Token Utilization (w:0.15). Her biri 0–100 normalize edildikten sonra weighted average. Çarpımlı ceza: loop_count > 10 veya retry_waste > %20 olduğunda sonuç ×0.85 ile çarpılır — yüksek döngü sayısı diğer iyi skorların arkasına saklanamaz.
Bileşik Formül SQI = Σ(wᵢ × scoreᵢ) × penalty → 0–100
penalty = 0.85 eğer loop_count>10 ∨ retry_waste>20%

JSONL Satır Şeması (Claude Code)
// ~/.claude/projects/<proje>/session-<id>.jsonl — her satır:
{
  "type": "assistant" | "user" | "system",
  "message": {
    "role": string,
    "content": [
      { "type": "text", "text": string },
      { "type": "tool_use", "name": "Read", "input": { "file_path": string } },
      { "type": "tool_result", "content": string }
    ]
  },
  "usage": {
    "input_tokens": number,  // ← CC Quality'nin ana veri kaynağı
    "output_tokens": number,
    "cache_read_input_tokens": number,
    "cache_creation_input_tokens": number
  },
  "timestamp": ISO 8601,
  "sessionId": string
}
